# Kovaaks UE4SS Mod Shim

This module is the UE4SS-facing shim. It loads `kovaaks_rust_core.dll` and forwards events to it.

## Build inputs

- UE4SS C++ SDK headers (`UE4SS_SDK_DIR`)
- Rust core DLL from `../ue4ss-rust-core`
- UE4SS import library for linking:
  - preferred: `ue4ss-mod/third_party/UE4SS.lib` (auto-generated by `scripts/dev-pipeline.sh` from runtime `UE4SS.dll`)
  - or a compatible SDK-provided import library

## Build

```bash
cmake -S . -B build -DUE4SS_SDK_DIR="C:/path/to/ue4ss/cppsdk"
cmake --build build --config Release
```

Output mod binary is `main.dll`.

For full repo orchestration (setup + rust core + mod + payload staging + app build), use:

- Windows: `pnpm run pipeline:win`
- Linux/WSL: `pnpm run pipeline:wsl` (Windows target, dev app build)
- SDK path is auto-detected from `external/` by the pipeline scripts.
- If `external/UE4SSCPPTemplate` exists, the pipeline will also try to initialize its submodules automatically.
- On WSL, only the C++ mod compile runs via Windows MSVC; Bash still orchestrates the overall pipeline.
- C++ compilation still requires UE4SS Unreal headers (`UEPseudo`), which are private and require Epic-linked GitHub access.

## Packaging layout

Place files in game directory:

```text
<game>/Binaries/Win64/
  UE4SS.dll
  ... UE4SS runtime files ...
  kovaaks_rust_core.dll
  Mods/
    KovaaksBridgeMod/
      mod.json
      dlls/
        main.dll
```

`mod.json` minimum example:

```json
{
  "mod_name": "KovaaksBridgeMod",
  "mod_type": "CppMod",
  "version": "0.1.0"
}
```

## Overlay-managed deployment

This project now ships UE4SS as a bundled overlay resource under:

- `src-tauri/ue4ss`

On overlay startup, the Tauri backend:

1. Syncs `src-tauri/ue4ss` payload into KovaaK's `Binaries/Win64`
2. Manually injects `UE4SS.dll` into `FPSAimTrainer-Win64-Shipping.exe`

So if the overlay is not started, the mod is not loaded.

To refresh payload files before packaging a new overlay release:

PowerShell (Windows):

```powershell
pwsh ./scripts/sync-ue4ss-payload.ps1 `
  -RuntimeDir "<extracted-ue4ss-runtime-dir>" `
  -ModMainDll "<path-to-main.dll>" `
  -RustCoreDll "<path-to-kovaaks_rust_core.dll>" `
  -RuntimeProfile minimal
```

Linux / WSL (Bash):

```bash
./scripts/sync-ue4ss-payload.sh \
  --runtime-dir "<extracted-ue4ss-runtime-dir>" \
  --mod-main-dll "<path-to-main.dll>" \
  --rust-core-dll "<path-to-kovaaks_rust_core.dll>" \
  --runtime-profile minimal
```

`minimal` is the default and excludes stock UE4SS sample mods/custom game configs.
Use `full` only if you intentionally want the entire upstream runtime tree.
Profiles also select UE4SS settings templates:
- `minimal` => production settings (`GraphicsAPI=dx11`, `EngineVersionOverride=4.27`, debug tooling off)
- `full` => development settings (`GraphicsAPI=dx11`, console/tooling enabled)

## Forwarded events (current)

- `score`, `kills`, `shots_hit`, `shots_fired`, `seconds`
- `damage_done`, `damage_possible`, `challenge_seconds`, `challenge_tick_count`
- `shot_fired`, `shot_hit`, `shot_missed`, `kill`
- `challenge_queued`, `challenge_start`, `challenge_complete`, `challenge_canceled`, `challenge_quit`, `challenge_restart`, `challenge_completed`

All events are emitted as newline-delimited JSON through `kovaaks_rust_core.dll` to `\\.\pipe\kovaaks-bridge`.

## Notes

- This repo does not vendor UE4SS SDK files.
- Runtime users only need UE4SS runtime + your mod DLLs, not your dump tooling.
- Gameplay hooks are implemented in `src/KovaaksBridgeMod.cpp` via a UE4SS `ProcessEvent` post-callback.
